```text
> Создание Spring-компонентов:
  ---------------------------
  1. Spring-компонентом может быть класс или класс наследуемый от интерфейса...
     Такой класс можно либо обозначить аннотацией (@Component, @Repository, @Service, @Controller, ...) либо можно НЕаннотировать...

> Конфигурирование Spring-компонентов:
  -----------------------------------
  1. Сконфигурировать все Spring-компоненты можно в xml-файле ресурсах приложения (для этого указать пакет где искать Spring-компоненты...при этом над классами должна стоять анотация @Component, @Repository, @Service, @Controller, ...чтобы Spring понимал что класс является Spring-компонентом)
  2. Сконфигурировать все Spring-компоненты можно в стиле-Java, для этого нужно над классом поставить анотацию @Configuration а внутри этого класса создать бины через оператор `new` с аннотацией @Bean (при этом сам класс-компонент можно НЕаннотировать...потому-что здесь мы явно указываем Spring-у что класс является Spring-компонентом)

> Получить Spring-компонент:
  -------------------------
  1. Если Spring-компоненты были описаны через xml-файл, тогда нужно с помощью аннотаций @EnabkeWebMvc, @EnableJpaRepositories, @EnableTransactionManagement, ...сказать Spring-у что нужно нужно получить этот компонент (поскольку этот компонент находится на самой вершине...)
     Все другие Spring-компоненты которые используются внутри этого класса можно получить через аннотацию @Autowired
  2. Можно с помощью классов имплементируемых интерфейс `AnnotationContext` .getBean(..)
  3. Если это веб-приложение которое имеет web.xml - в этом случае можно получения Spring-бинов описать внутри этого web.xml
```
